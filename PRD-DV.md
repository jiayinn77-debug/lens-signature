# 模拟 DV 镜头签 App PRD

## 1. 产品概述

### 1.1 产品名称

镜头签

### 1.2 一句话定位

一款可以切换不同复古 DV / CCD / 胶片相机型号的录像 App。用户只能通过实时录制生成内容，并在录制过程中留下动态镜头签。录制完成后，系统保存一段复古视频，并从视频中自动抽出三张好看的关键帧照片。

### 1.3 产品核心

镜头签不是照片编辑器，也不是导入照片后加滤镜的工具。

它的核心是：

> 选择一台复古相机，录下一段带有专属镜头签的视频，并自动获得三张像胶片截图一样的关键帧。

### 1.4 核心产物

每次创作完成后，用户获得：

- 1 段复古滤镜视频
- 3 张从视频中抽出的关键帧照片

产品不支持导入照片，不支持对已有照片做后期编辑。所有内容必须来自实时录制。

## 2. 产品定位

### 2.1 产品类型

模拟 DV / 复古相机录像 App。

### 2.2 差异点

与普通复古相机 App 的区别：

- 普通 App：选择滤镜，拍照或录像。
- 镜头签：选择相机型号，录制一段视频，并在视频过程中留下动态签名。

### 2.3 体验关键词

- 复古 DV
- CCD
- 老相机
- 实时录制
- 动态镜头签
- 过程感
- 仪式感
- 1 段视频 + 3 张照片

## 3. 目标用户

### 3.1 核心用户

- 喜欢 CCD、DV、胶片、宝丽来风格的用户
- 喜欢用短视频记录生活片段的用户
- 喜欢发小红书、朋友圈、Instagram、抖音的用户
- 想让普通视频拥有个人印记的人
- 喜欢复古机器感和录制仪式的人

### 3.2 用户需求

- 我想用不同复古相机风格录视频。
- 我想让视频看起来像老 DV、CCD 或胶片机拍出来的。
- 我想在录制时留下一个属于自己的镜头签。
- 我想录完后直接得到视频和几张好看的截图。
- 我不想复杂后期，只想边录边生成。

## 4. 核心产品原则

### 4.1 只录制，不导入

产品只支持实时录制，不支持导入照片或视频进行编辑。

原因：

- 保持产品定位清晰。
- 强化“这是一台相机”，不是“后期编辑器”。
- 让镜头签成为录制过程的一部分。
- 让最终内容更真实、更有现场感。

### 4.2 先选择相机型号，再开始录制

用户进入 App 后，不是先进入编辑器，而是先选择一台模拟相机。

每台相机决定：

- 取景器 UI
- 视频滤镜
- 噪点和颗粒
- 水印样式
- 录制声音
- 按钮样式
- 镜头签笔触
- 导出成片外观

### 4.3 镜头签必须是动态过程

签名不是录完后贴上去的静态图层，而是在录制过程中出现的动态轨迹。

用户可以通过：

- 手势签名
- 触屏签名

在画面中留下签名过程。

### 4.4 输出必须天然适合分享

录制结束后，系统自动生成：

- 视频：适合短视频平台
- 关键帧：适合图文平台、封面、朋友圈

## 5. 核心流程

### 5.1 标准用户流程

```text
打开 App
-> 选择模拟相机型号
-> 进入该型号的取景器界面
-> 点击录制
-> 拍摄真实画面
-> 在录制过程中完成镜头签
-> 停止录制
-> 系统生成视频
-> 系统自动抽取三张关键帧
-> 用户预览视频和照片
-> 保存到相册
```

### 5.2 MVP 流程

```text
选择相机 -> 开始录制 -> 签名 -> 停止录制 -> 预览 -> 保存视频和三张照片
```

### 5.3 不支持流程

MVP 和当前产品方向不支持：

- 导入照片
- 导入已有视频
- 对相册素材做后期编辑
- 只保存静态照片而不录视频

## 6. 功能模块

## 6.1 模拟相机型号选择

### 6.1.1 功能目标

把传统“滤镜选择”升级为“选择相机型号”。用户感觉自己是在换一台机器，而不是套一个效果。

### 6.1.2 相机型号定义

每台相机包含：

- 相机名称
- 取景器样式
- 视频画质
- 滤镜
- 边框或水印
- 噪点、颗粒、漏光
- 快门/录制声音
- 按钮外观
- 镜头签默认笔触
- 导出模板

### 6.1.3 首批相机型号建议

#### CCD-01

- 风格：早期 CCD 数码相机
- 画质：高噪点、轻微过曝、闪光灯感
- UI：小屏幕取景器、日期水印、电池图标
- 签名：白色发光细线
- 适合：朋友聚会、夜晚、生活碎片

#### DV-98

- 风格：家用 DV 录像机
- 画质：低清、轻微拖影、时间戳
- UI：REC 红点、磁带计时、取景框
- 签名：像素化白字轨迹
- 适合：日常记录、旅行、街景

#### HANDYCAM

- 风格：掌上录像机
- 画质：低饱和、轻微抖动、模拟扫描线
- UI：磁带余量、REC 状态、变焦条
- 签名：黄色日期水印风格
- 适合：家庭感、回忆感、室内场景

#### POLA-SX

- 风格：宝丽来相机
- 画质：柔和、显影感、奶白边框
- UI：相纸吐片、显影进度
- 签名：手写白墨或烫金笔触
- 适合：人像、静物、纪念瞬间

#### TOY CAM

- 风格：玩具相机
- 画质：色偏、暗角、塑料镜头感
- UI：彩色按钮、简化取景框
- 签名：彩色涂鸦轨迹
- 适合：可爱、轻松、随机感内容

## 6.2 复古相机身体 UI

### 6.2.1 功能目标

每个相机型号都应该拥有自己的“身体”，让用户感觉是在操作不同设备。

### 6.2.2 主要区域

- 顶部状态区
  - 当前相机型号
  - REC 状态
  - 录制时间
  - 电量或磁带余量装饰
  - 切换前后摄像头

- 中间取景区
  - 实时摄像头画面
  - 相机型号对应滤镜
  - 镜头签轨迹
  - 取景框、日期、水印、参数

- 底部操作区
  - 大号录制按钮
  - 镜头签模式按钮
  - 撤回按钮
  - 切换相机按钮
  - 保存入口

### 6.2.3 操作命名

| 普通功能 | 产品化命名 |
| --- | --- |
| 选择滤镜 | 换相机 |
| 开始录制 | REC |
| 停止录制 | STOP |
| 保存 | 存入相册 |
| 撤回 | 倒带 |
| 清空签名 | 重签 |
| 预览 | 回放 |
| 导出 | 出片 |

## 6.3 实时录制

### 6.3.1 功能目标

用户通过手机摄像头实时录制内容，录制期间自动应用所选相机型号的画质和界面效果。

### 6.3.2 录制要求

- 支持前后摄像头切换。
- 录制过程中显示 REC 状态。
- 录制过程中显示录制时长。
- 录制过程中应用相机型号滤镜。
- 录制过程中记录镜头签轨迹。
- 停止录制后进入回放预览。

### 6.3.3 录制时长

MVP 建议：

- 最短：3 秒
- 最长：15 秒
- 推荐：5 到 8 秒

较短的视频更适合生成关键帧和社交传播。

## 6.4 镜头签

### 6.4.1 功能目标

用户在录制过程中留下动态签名，让视频有个人印记。

### 6.4.2 输入方式

MVP 支持：

- 触屏签名
- 手势签名

### 6.4.3 签名出现方式

签名必须作为视频过程的一部分被录制下来。

可支持：

- 边录边签
- 签名轨迹实时出现
- 笔迹有轻微发光、颗粒、色散或拖影

### 6.4.4 签名风格

不同相机型号可以绑定不同签名风格：

| 相机型号 | 默认签名风格 |
| --- | --- |
| CCD-01 | 白色发光签名 |
| DV-98 | 像素化签名 |
| HANDYCAM | 黄色时间戳风格签名 |
| POLA-SX | 白墨或烫金手写 |
| TOY CAM | 彩色涂鸦签名 |

## 6.5 声音和触感

### 6.5.1 功能目标

通过声音和触感增强设备真实感。

### 6.5.2 声音反馈

| 场景 | 声音 |
| --- | --- |
| 切换相机型号 | 机械拨盘声 |
| 开始录制 | DV 启动声 / REC 哔声 |
| 停止录制 | 磁带停止声 |
| 签名落笔 | 轻微电流声 / 书写声 |
| 保存成功 | 相册保存提示声 |
| 倒带/撤回 | 磁带倒带声 |

### 6.5.3 触感反馈

原生 App 阶段实现：

- 点击 REC 短震
- 停止录制短震
- 切换相机型号短震
- 保存成功轻震
- 签名落笔时轻微连续反馈

### 6.5.4 设置

- 提供静音开关。
- 声音应该偏复古、小机器感，避免刺耳。

## 6.6 视频生成与保存

### 6.6.1 功能目标

录制结束后生成一段完整视频，并保存到系统相册。

### 6.6.2 视频内容

视频应包含：

- 摄像头画面
- 所选相机滤镜
- 相机型号 UI 元素
- REC / 时间戳 / 日期等水印
- 动态镜头签过程
- 颗粒、噪点、漏光、暗角等风格层

### 6.6.3 视频格式

网页 demo 阶段：

- 优先支持浏览器可录制格式，例如 WebM。
- 如 iOS Safari 受限，需要明确提示或使用替代方案。

App 阶段：

- 优先导出 MP4。
- 保存到系统相册。
- 支持分享到社交平台。

## 6.7 三张关键帧照片

### 6.7.1 功能目标

录制结束后，系统从视频中自动抽取三张好看的照片并保存到相册。

### 6.7.2 三张照片定义

#### 第 1 张：开场氛围帧

- 视频前段画面。
- 签名前或刚开始签名。
- 展示相机型号画质和氛围。

#### 第 2 张：签名过程帧

- 视频中段画面。
- 签名正在出现。
- 最能体现产品特色。

#### 第 3 张：最终成片帧

- 视频末段画面。
- 签名完整。
- 画面、水印、滤镜稳定。
- 适合作为封面或社交图文第一张。

### 6.7.3 MVP 抽帧策略

按时间点抽帧：

- 第 1 张：视频 15% 位置
- 第 2 张：视频 55% 位置
- 第 3 张：视频 90% 位置

### 6.7.4 后续智能抽帧

后续可升级：

- 避免模糊帧
- 避免画面过暗或过曝
- 优先选择签名清晰帧
- 支持用户手动替换关键帧
- 支持选择其中一张作为视频封面

## 9. 设计方向

### 9.1 视觉关键词

- 复古 DV
- CCD
- 家用录像机
- 取景器
- 磁带
- REC 红点
- 时间戳
- 老数码屏幕
- 颗粒和噪点
- 真实小机器

### 9.2 UI 建议

- 主界面像一台真实设备，而不是普通编辑器。
- 不同相机型号有不同外观。
- 按钮要有按压感。
- REC 状态要明显。
- 时间戳和水印要融入画面，不要像后贴上去的 UI。
- 签名轨迹要和相机画质融合。

### 9.3 避免方向

- 避免照片编辑器式布局。
- 避免导入照片入口。
- 避免过多参数调节。
- 避免普通滤镜列表。
- 避免现代感过强的 UI。

## 10. 成功指标

### 10.1 核心体验指标

- 用户是否能快速理解“选相机 -> 录视频 -> 签名 -> 得到视频和 3 张照片”。
- 用户是否能在 1 分钟内完成一次录制。
- 用户是否觉得每个相机型号有明显差异。
- 用户是否愿意保存最终视频和照片。

### 10.2 内容指标

- 视频是否有复古相机质感。
- 签名过程是否足够明显。
- 三张关键帧是否适合发布。
- 最终结果是否有社交分享欲。

### 10.3 后续数据指标

- 录制完成率
- 保存率
- 分享率
- 重录率
- 相机型号切换次数
- 用户平均生成视频数量

## 11. 技术备注

### 11.1 当前 demo 可复用能力

- 摄像头调用
- 前后摄像头切换
- canvas 签名轨迹
- 手势识别
- 触屏签名
- MediaRecorder 录制
- 基础滤镜

### 11.2 需要调整的方向

- 移除导入照片相关设计和入口。
- 将“胶卷选择”升级为“相机型号选择”。
- 将“保存成片”升级为“保存视频 + 三张关键帧”。
- 将“编辑工具 UI”改为“模拟相机身体 UI”。

### 11.3 风险点

- iOS Safari 对 MediaRecorder 和视频格式支持有限。
- 手机浏览器保存到相册能力有限，正式 App 阶段体验会更完整。
- 实时滤镜、签名和录制同时运行可能有性能压力。
- 手势识别在弱光环境下可能不稳定。

### 11.4 技术建议

网页 demo 阶段：

- 优先跑通录制流程。
- 视频可先导出为浏览器支持的格式。
- 三张关键帧可先用 canvas 导出 PNG。
- 保存到相册如受限，可以先提供下载，再在 App 阶段做系统相册保存。

App 阶段：

- 使用 MP4 导出。
- 接入系统相册保存。
- 增加触感反馈。
- 优化实时滤镜性能。
- 增加更稳定的关键帧抽取。
