# 镜头签 PRD

## 1. 产品概述

### 1.1 产品名称

镜头签

### 1.2 一句话定位

一款把手机变成复古相机的互动影像工具，用户在拍摄前选择胶卷风格，通过手势或触屏完成镜头签，最终导出一段复古滤镜签名视频和从视频中抽出的三张关键帧。

### 1.3 产品核心

镜头签不是普通修图工具，也不是单纯给照片加水印。它的核心是：

> 把签名变成一段复古影像生成过程。

用户最终得到的不是一张静态图片，而是一段带有仪式感的影像：

- 拍摄或导入画面
- 选择一卷有性格的胶卷
- 用手势或触屏留下签名轨迹
- 录制签名出现的过程
- 生成一段复古视频
- 自动抽出三张适合社交发布的关键帧

### 1.4 当前阶段

当前项目已有手机版网页 demo，已具备以下基础能力：

- 摄像头调用
- 前后摄像头切换
- 手势识别签名
- 触屏签名
- 基础滤镜
- 笔触控制
- 录制导出

下一阶段目标是从“功能 demo”升级为“有产品气质的互动软件 demo”。

## 2. 产品目标

### 2.1 MVP 目标

完成一个可在手机浏览器中体验的复古相机式镜头签 demo，让用户可以完整走通：

```text
选择胶卷 -> 拍摄/导入 -> 签名 -> 录制签名过程 -> 生成视频 -> 抽取三张关键帧 -> 保存/分享
```

### 2.2 体验目标

- 用户一打开就感觉自己在操作一台复古小相机。
- 胶卷选择不是普通滤镜选择，而是拍摄前的仪式。
- 签名不是结果，而是一个被记录下来的动态过程。
- 导出内容天然适合小红书、朋友圈、Instagram、抖音等平台传播。
- 用户愿意反复尝试不同胶卷和不同签名效果。

### 2.3 产品化目标

后续可以从手机版网页 demo 迁移为手机 App，支持更完整的相册权限、视频导出、触感反馈、本地作品库和社交分享。

## 3. 目标用户

### 3.1 核心用户

- 喜欢拍照、修图、发社交平台的年轻用户
- 喜欢 CCD、胶片、宝丽来、复古相机风格的人
- 想让普通照片拥有个人印记的人
- 喜欢互动式创作，而不是只套模板的人
- 喜欢“过程感”和“仪式感”的影像创作者

### 3.2 用户需求

- 我想让普通画面变得更有复古感和设计感。
- 我想拥有一个属于自己的照片或视频签名方式。
- 我想导出可以直接发布到社交平台的内容。
- 我想让生成过程本身也好看，而不是只得到最终图片。
- 我希望这个工具像一台有性格的小相机，而不是普通编辑器。

## 4. 产品主流程

### 4.1 标准流程

```text
打开镜头签
-> 进入胶卷选择页
-> 选择胶卷
-> 进入复古相机界面
-> 拍摄视频/照片或从相册导入
-> 进入镜头签模式
-> 用户用手势或触屏完成签名
-> 系统录制签名过程
-> 生成复古签名视频
-> 自动抽取三张关键帧
-> 用户预览视频和关键帧
-> 保存或分享
```

### 4.2 MVP 推荐流程

为了降低第一版复杂度，MVP 可以先聚焦这一条路径：

```text
选胶卷 -> 开摄像头 -> 开始录制 -> 完成签名 -> 停止录制 -> 生成视频和三张关键帧
```

相册导入、更多导出比例、自动封面选择可以作为 P1 或 P2。

## 5. 核心功能

## 5.1 复古相机身体 UI

### 5.1.1 功能目标

把普通网页控制面板改造成一台复古相机，让用户有“拿起相机拍摄”的感觉。

### 5.1.2 界面结构

主界面建议包含：

- 顶部状态区
  - 品牌名或产品名
  - 当前胶卷名称
  - 拍摄编号
  - 电量/胶卷剩余张数装饰
  - 切换摄像头按钮

- 中间取景区
  - 摄像头画面
  - 签名轨迹
  - 对焦框或取景框
  - 当前状态提示

- 底部相机控制区
  - 大号快门/录制按钮
  - 胶卷入口
  - 撤回按钮
  - 保存/冲洗按钮
  - 图库入口

### 5.1.3 命名包装

将普通功能命名包装为相机语言：

| 普通功能 | 产品化命名 |
| --- | --- |
| 滤镜 | 胶卷 |
| 保存 | 冲洗 |
| 录制 | 开拍 |
| 停止录制 | 收片 |
| 撤回 | 倒片 |
| 清空 | 重拍 |
| 预览 | 成片 |
| 导出 | 出片 |

### 5.1.4 交互要求

- 快门按钮需要有明确的按压反馈。
- 录制状态需要有红点、计时或胶卷计数变化。
- 胶卷切换需要有滑动或卡入感。
- 控制按钮尽量图形化、机械化、拟物化。
- 页面不能像表单工具，应该像相机设备。

## 5.2 拍摄前选择胶卷

### 5.2.1 功能目标

把“选择滤镜”前置为一次拍摄仪式。用户不是调参数，而是在选择今天要装进相机里的胶卷。

### 5.2.2 胶卷定义

每卷胶卷是一组风格预设，包括：

- 画面滤镜
- 颗粒和噪点
- 漏光或闪烁
- 签名笔触风格
- 水印文字
- 导出模板
- 声音反馈

### 5.2.3 首批胶卷建议

#### Expired 200

- 风格：过期胶片
- 视觉：偏暖、褪色、颗粒明显、轻微漏光
- 适合：日常、街景、旅行、旧照片感

#### Night Bloom

- 风格：夜间 CCD
- 视觉：偏蓝绿、闪光灯、噪点重、高光溢出
- 适合：夜晚、朋友聚会、室内闪光灯

#### Soft Morning

- 风格：清晨柔光
- 视觉：低对比、奶油色、高光柔化、轻雾感
- 适合：早餐、窗边、天空、生活碎片

#### Room Light

- 风格：室内生活感
- 视觉：暖光、轻微模糊、低饱和、真实日常
- 适合：房间、咖啡馆、餐桌、宠物

#### Memory Film

- 风格：回忆胶片
- 视觉：低饱和、边缘暗角、轻微褪色
- 适合：人像、旅行、旧物、情绪照片

### 5.2.4 胶卷卡片信息

每张胶卷卡片包含：

- 胶卷名称
- 预览图
- 风格标签
- 剩余张数装饰，例如 `12 shots`
- 简短描述，例如 `warm grain / soft fade`

## 5.3 镜头签互动

### 5.3.1 功能目标

让用户通过手势或触屏在画面上留下个人化签名，并且记录签名出现的全过程。

### 5.3.2 输入方式

MVP 支持：

- 手势签名
- 触屏签名

后续可支持：

- 文字签名模板
- 贴纸式签名
- 签名轨迹二次编辑

### 5.3.3 签名风格

首批签名风格建议：

- 白墨签名
- 发光签名
- CCD 色散签名
- 胶片颗粒签名
- 烫金签名

### 5.3.4 签名控制

MVP 控制项：

- 粗细
- 颜色或签名预设
- 发光开关
- 撤回
- 清空

建议第一版减少复杂参数，优先提供 3 到 5 个好看的签名预设。

## 5.4 声音和触感

### 5.4.1 功能目标

增强相机操作的真实感，让用户感到自己在操作一台小机器。

### 5.4.2 声音反馈

MVP 可先实现声音：

| 场景 | 声音 |
| --- | --- |
| 按下快门 | 咔嚓声 |
| 开始录制 | 轻微机械启动声 |
| 停止录制 | 胶卷收片声 |
| 切换胶卷 | 胶卷滑动声 |
| 保存成功 | 冲洗完成声 |
| 撤回 | 倒片声 |

### 5.4.3 触感反馈

原生 App 阶段实现：

- 快门按钮短震
- 胶卷切换短震
- 保存成功轻震
- 签名落笔时轻微连续反馈

### 5.4.4 设置要求

- 提供静音开关。
- 声音不能刺耳，要偏温柔、复古、小机器感。
- 网页 demo 可先用轻量音效，App 阶段再精细化。

## 5.5 签名视频导出

### 5.5.1 功能目标

用户最终获得一段 3 到 8 秒的复古镜头签视频。视频记录签名出现、滤镜显影、画面定格的过程。

### 5.5.2 视频结构

建议视频由四段组成：

#### 0-1 秒：取景/显影

- 画面从轻微模糊、曝光闪烁中出现。
- 胶卷滤镜逐渐加载。
- 可出现当前胶卷名称和拍摄编号。

#### 1-4 秒：签名过程

- 用户通过手势或触屏完成签名。
- 笔迹真实出现在画面上。
- 可叠加轻微颗粒、漏光、闪烁。

#### 4-6 秒：成片定格

- 签名完成。
- 滤镜、边框、水印、参数稳定出现。
- 画面像一张完成的胶片作品。

#### 最后 0.5 秒：冲洗完成

- 轻微闪光或机械反馈。
- 适合视频结尾循环。

### 5.5.3 视频内容元素

视频可以包含：

- 原始画面或实时拍摄画面
- 胶卷滤镜
- 签名轨迹
- 日期
- 胶卷名
- 虚拟相机参数
- 拍摄编号
- 轻微颗粒、漏光、暗角、闪烁

示例参数：

```text
35MM / F2.8 / ISO 200
MEMORY FILM / TAKE 03
SHOT ON 2026.07.12
```

## 5.6 三张关键帧

### 5.6.1 功能目标

从签名视频中自动抽取三张好看的关键帧，作为社交图文发布素材。

### 5.6.2 三张关键帧定义

#### Frame 1：氛围图

- 签名前或签名刚开始前的画面。
- 展示胶卷滤镜和画面氛围。
- 适合作为图文第一张或预览图。

#### Frame 2：过程图

- 签名进行中的画面。
- 有动态感和过程感。
- 适合展示产品特色。

#### Frame 3：最终成片

- 签名完整。
- 构图稳定。
- 水印、参数、滤镜完整。
- 适合作为封面。

### 5.6.3 抽帧策略

MVP 可以按时间点抽帧：

- 第 1 张：视频 15% 位置
- 第 2 张：视频 55% 位置
- 第 3 张：视频 90% 位置

后续可升级为智能抽帧：

- 避免模糊帧
- 避免眨眼或明显糊掉的画面
- 优先选择签名清晰、构图稳定的帧
- 支持用户手动替换关键帧

## 5.7 社交传播包

### 5.7.1 功能目标

导出时不只给用户一个视频文件，而是生成一组可以直接发布的内容包。

### 5.7.2 内容包组成

MVP 社交包：

- 1 段复古签名视频
- 3 张关键帧图片
- 1 张最终成片图

后续可扩展：

- 自动封面图
- 小红书标题建议
- 发布文案
- 胶片参数说明图
- 9:16 竖屏视频版
- 3:4 图文版

### 5.7.3 导出尺寸

优先支持：

- 9:16：适合抖音、Reels、Story
- 3:4：适合小红书图文
- 4:5：适合 Instagram、小红书

MVP 可以先支持当前取景比例，后续再做比例选择。

## 8. 设计方向

### 8.1 关键词

- 复古
- 胶片
- CCD
- 宝丽来
- 小机器
- 仪式感
- 私人印记
- 不完美
- 过程感
- 可传播

### 8.2 视觉建议

- 主色：奶白、银灰、黑色、暗红、胶片黄
- 材质：磨砂塑料、金属按钮、玻璃取景框
- 字体：参数信息使用等宽字体，品牌标题可使用复古 serif
- 按钮：有凹凸、阴影、按压状态
- 画面：保留颗粒、轻微漏光、轻微暗角

### 8.3 避免方向

- 过于普通的现代工具栏
- 大量表单控件
- 科技感过强的蓝紫渐变
- 参数过多导致用户不知道怎么选
- 只好看但不符合真实操作流程的 UI

## 9. 成功指标

### 9.1 体验指标

- 用户是否能在 1 分钟内完成一段签名视频。
- 用户是否理解“选胶卷 -> 签名 -> 出片”的流程。
- 用户是否觉得界面像一台复古相机。
- 用户是否愿意尝试不同胶卷。
- 用户是否愿意保存视频或关键帧。

### 9.2 内容指标

- 生成的视频是否有明显过程感。
- 三张关键帧是否适合直接发布。
- 最终成片是否有复古相机气质。
- 胶卷风格之间是否有明显差异。

### 9.3 后续产品指标

- 完成率
- 保存率
- 分享率
- 重复使用率
- 胶卷切换次数
- 用户平均生成内容数量

## 10. 技术备注

### 10.1 当前网页 demo 可延续能力

- 使用摄像头画面作为取景层。
- 使用 canvas 绘制签名轨迹。
- 使用 MediaRecorder 录制 canvas 合成画面。
- 使用 canvas 在指定时间点导出关键帧。
- 使用 CSS filter 或 canvas filter 实现基础胶卷效果。

### 10.2 关键技术任务

- 将现有控制面板重构为复古相机布局。
- 将滤镜选择改造成胶卷选择。
- 在录制过程中合成视频画面、签名层、水印层和滤镜层。
- 录制结束后自动抽取三张关键帧。
- 增加音效播放和静音开关。

### 10.3 风险点

- iOS Safari 对视频格式和 MediaRecorder 支持有限。
- 手机浏览器访问摄像头可能需要 HTTPS。
- 实时滤镜和录制可能影响性能。
- 手势签名在不同光线和背景下稳定性不同。
- 视频导出格式需要兼容社交平台。

### 10.4 技术优先建议

网页 demo 阶段：

- 优先保证流程跑通。
- 视频可以先导出 WebM。
- 关键帧可以先用 canvas 导出 PNG。
- 胶卷滤镜可以先用 CSS filter 加轻量叠层。

App 阶段：

- 优先解决 MP4 导出。
- 增加系统相册保存。
- 增加触感反馈。
- 增加更稳定的视频处理和本地作品库。
